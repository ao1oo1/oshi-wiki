# Oshi-Wiki v2：AI執筆補助機能

## v2の目的

v2では、AI本文生成は実装せず、AIに貼り付けるためのプロンプト作成・保存・コピー機能を実装する。

## v2で実装すること

- 一般ユーザー登録
- 一般ユーザーログイン
- 一般ユーザー用マイページ
- オリジナルキャラクター登録
- オリジナルキャラクター関係性登録
- v1の作品・キャラクター・関係性データ参照
- v1データとv2データを組み合わせたプロンプト作成
- プロンプト保存
- プロンプトコピー

## v2では実装しないこと

- OpenAI API連携
- Ollama連携
- サイト内AI本文生成
- AIによる自動執筆

## DB使用制限

1ユーザーあたり：

- オリジナルキャラクター：最大30件
- 関係性：最大100件
- 保存プロンプト：最大50件
- 1プロンプト本文：最大20,000文字
- あらすじ：最大5,000文字
- 各備考：最大2,000〜5,000文字

## 実装フェーズ

1. v2ブランチ作成・v1タグ確認
2. 一般ユーザー登録/ログイン/マイページ
3. DB使用制限の共通定数作成
4. オリジナルキャラクターDB/CRUD
5. 関係性DB/CRUD
6. プロンプト作成DB/CRUD
7. v1データ抽出
8. プロンプト生成ロジック
9. コピー機能・UI調整・v2タグ保存

### 最高管理者の制限

v2のDB使用制限は一般ユーザー向けの制限とする。

最高管理者権限保持者は、以下の制限を受けない。

- オリジナルキャラクター登録数
- 関係性登録数
- 保存プロンプト数
- プロンプト本文文字数
- あらすじ文字数
- 備考文字数

実装上は、`App\Support\WritingAssistLimits` でユーザーを受け取り、最高管理者の場合は `null` を返す。  
`null` は「制限なし」として扱う。

## Phase 6：保存プロンプト

AIに直接本文生成させる機能は実装せず、ユーザーがAIに貼り付けるためのプロンプトを保存・編集・コピーできるようにする。

### 保存項目

- タイトル
- カテゴリ
- 用途
- あらすじ・前提
- プロンプト本文
- 備考
- 状態

### 制限

一般ユーザーは保存プロンプトを最大50件まで保存できる。  
最高管理者は保存件数・文字数ともに制限なし。

### コピー機能

詳細画面に「本文をコピー」ボタンを設置し、保存済みプロンプト本文をクリップボードへコピーできるようにする。

## Phase 7：登録済みデータを使ったプロンプト精度強化

保存プロンプト作成時、選択されたキャラクターの登録済み情報をプロンプト本文へ自動反映する。

### 反映対象

- v1 作品キャラクター情報
- v2 オリジナルキャラクター情報
- v1 作品キャラクター同士の関係性
- v2で登録した関係性

### 目的

ユーザーが都度キャラクター設定を手入力しなくても、登録済みデータベースを元にAIへ渡すプロンプトを作成できるようにする。

## Phase 8：プロンプト作成画面の使いやすさ改善

プロンプト作成画面を、項目ごとに入力しやすいステップ形式へ改善する。

### 改善内容

- 作品選択に応じて、作品キャラクターを絞り込み表示する
- オリジナルキャラクターと作品キャラクターを分けて表示する
- 登場人物選択を複数選択ボックスからチェックボックス形式へ変更する
- 作風「その他」選択時のみ自由入力欄を表示する
- ジャンル「その他」選択時のみ自由入力欄を表示する
- 入力補助文を追加する
