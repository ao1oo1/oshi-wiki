# OshiBase — 創作支援データベース（MVP / Phase 1）

二次創作・夢小説を書く人向けに、キャラクターの一人称・二人称・呼称・口調・関係性を出典付きで整理するデータベースサイトです。

設計書の Phase 1（最小公開版）＋ 管理者ログイン・権限・承認フローまでを実装しています。

## 技術構成（設計書からの変更点）

設計書では Next.js + Firebase/Vercel が候補でしたが、**さくらのレンタルサーバでは Node.js アプリを常時稼働できない**ため、さくらで確実に動く構成に変更しています。

| 項目 | 採用技術 | 理由 |
|---|---|---|
| 言語 | PHP 8（フレームワークなし） | さくら標準対応。FTPアップロードだけで動く |
| DB | SQLite（初期）→ MySQL（切替可） | ローカルはゼロ設定。さくらではどちらも利用可 |
| CSS | 自作CSS（設計書のカラーパレット準拠） | ビルド不要。CDN依存も最小 |
| フォント | Zen Maru Gothic + Noto Sans JP（Google Fonts） | 丸ゴシックで「可愛すぎず実用的」なトーン |

## 実装済み機能

**公開側（ログイン不要）**
- トップページ（大きな検索窓・人気作品・最近追加キャラ）
- 作品一覧 / 作品詳細（キャラ一覧・呼称表・用語・出典）
- キャラクター詳細（基本情報 / 口調・一人称 / 呼称一覧 / 関係性 / 出典 のタブ切替）
- 横断検索（作品名・キャラ名・一人称・呼称・語尾・用語）
- 情報提供・修正依頼フォーム（承認制 / 情報区分付き）
- OshiBaseについて / キュレーター募集ページ
- 削除依頼フォーム / 利用規約・ガイドライン

**管理側（ログイン必須）**
- ダッシュボード（承認待ち件数・最近の受付・編集履歴）
- 作品 / キャラクター（＋口調プロファイル） / 呼称 / 用語 / 出典 のCRUD
- フォーム受付の承認フロー（承認待ち → 確認済み → 反映済み / 見送り）
- 編集履歴（監査ログ）
- ユーザー・権限管理（最高管理者のみ）、担当作品（キュレーター）割り当て
- アカウント設定（自分のパスワード変更）

**権限（設計書 §8 準拠）**

| 操作 | 必要権限 |
|---|---|
| 管理画面の閲覧 | 閲覧スタッフ以上 |
| 受付の「確認済み」化 | 確認担当以上 |
| キャラ・呼称・用語・出典の編集 / 承認 | キュレーター以上（担当作品のみ） |
| 作品作成 / キャラ削除 | 管理者以上 |
| 作品削除 / 権限変更 / ユーザー管理 | 最高管理者のみ |

**セキュリティ**
- パスワードは `password_hash()`（bcrypt）で保存
- 全POSTにCSRFトークン必須
- 全出力を `htmlspecialchars` でエスケープ（XSS対策）
- 全クエリをプリペアドステートメント化（SQLi対策）
- ログイン5回失敗で60秒ロック / セッションID再生成
- `app/` `data/` は `.htaccess` で外部アクセス拒否

## 1. ローカルで動かす

PHP 8 以上（mbstring, pdo_sqlite 有効）が必要です。

```bash
cd oshibase
php -S localhost:8080
```

ブラウザで http://localhost:8080 を開くと、初回アクセス時にDB作成・サンプルデータ投入・初期管理者の作成が自動で行われます。

**初期管理者ログイン**（`/admin/login.php`）

- メール: `admin@example.com`
- パスワード: `oshibase-admin`

→ ログイン後、必ず「アカウント設定」または「ユーザー・権限管理」からパスワードを変更してください。
初期アカウント情報は `app/config.php` の `initial_admin` で変更できます（DB作成前に変更すれば反映されます）。

サンプルデータ（架空作品「星降学園物語」）が不要になったら、管理画面の作品管理から削除するか、`app/config.php` の `seed_sample_data` を `false` にして `data/oshibase.sqlite` を消してやり直してください。

## 2. さくらのレンタルサーバへのデプロイ

対応プラン: スタンダード以上を推奨（PHP・MySQL対応プラン）。

### 手順

1. **PHPバージョン設定**
   さくらのコントロールパネル → スクリプト設定 → PHPのバージョン設定 で **PHP 8.x（モジュール版推奨）** を選択。

2. **ファイルのアップロード**
   `oshibase` フォルダの中身一式を FTP/SFTP で `~/www/`（またはサブディレクトリ `~/www/oshibase/`）へアップロード。
   - `.htaccess`（ルート・`app/`・`data/`）も忘れずアップロードしてください。隠しファイル表示を有効に。

3. **data ディレクトリの書き込み権限**
   SQLiteのまま使う場合、`data/` ディレクトリをパーミッション `705` または `755`、SQLite作成後のファイルは `606` にします（さくらはユーザー権限でPHPが動くため通常はそのままで書き込めます）。

4. **動作確認**
   `https://アカウント名.sakura.ne.jp/`（または独自ドメイン）にアクセス → 初回アクセスでDBが自動作成されます。

5. **初期パスワード変更**（必須）
   `/admin/login.php` からログインし、パスワードを変更。

### MySQLに切り替える場合（推奨・アクセス増加時）

1. さくらコントロールパネル → データベース で MySQL を新規作成
2. `app/config.php` を編集:

```php
'db_driver' => 'mysql',
'mysql' => [
    'host'     => 'mysqlXXX.db.sakura.ne.jp',  // コンパネに表示されるホスト
    'dbname'   => 'アカウント名_oshibase',
    'user'     => 'アカウント名',
    'password' => 'DBパスワード',
    'charset'  => 'utf8mb4',
],
```

3. サイトに初回アクセスするとテーブルが自動作成されます。
   ※ SQLiteからのデータ移行が必要な場合は、両DBを繋いでコピーする簡単なスクリプトで対応できます（データ量が少ないうちの切替を推奨）。

### 独自ドメイン + SSL

1. お名前.com やさくらのドメインでドメイン取得
2. さくらコントロールパネル → ドメイン/SSL → ドメイン新規追加 で設定（さくら外で取得した場合はネームサーバーを `ns1.dns.ne.jp` / `ns2.dns.ne.jp` に変更）
3. 無料SSL（Let's Encrypt）を有効化
4. ドメイン設定で「HTTPSに転送する」を有効化

## 3. ディレクトリ構成

```
oshibase/
├── index.php          トップページ（検索窓）
├── works.php          作品一覧
├── work.php           作品詳細
├── character.php      キャラクター詳細（タブUI）
├── search.php         横断検索
├── about.php          OshiBaseについて
├── community.php      キュレーター募集
├── submit.php         情報提供・修正依頼フォーム
├── takedown.php       削除依頼フォーム
├── terms.php          利用規約・ガイドライン
├── assets/style.css   スタイル（設計書パレット準拠）
├── admin/             管理画面（要ログイン）
│   ├── login.php / logout.php
│   ├── index.php      ダッシュボード
│   ├── works.php      作品管理
│   ├── characters.php キャラ管理（口調プロファイル同時編集）
│   ├── appellations.php 呼称管理
│   ├── terms_admin.php  用語管理
│   ├── sources.php    出典管理
│   ├── submissions.php フォーム受付・承認フロー
│   ├── logs.php       編集履歴
│   ├── profile.php    アカウント設定
│   └── users.php      ユーザー・権限管理（最高管理者のみ）
├── app/               ライブラリ（Web非公開）
│   ├── config.php     設定（DB切替・初期管理者）
│   ├── db.php         DB接続・スキーマ・シード
│   └── helpers.php    認証・CSRF・レイアウト
└── data/              SQLite格納先（Web非公開）
```
